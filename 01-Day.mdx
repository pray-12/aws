## History of Cloud Computing
In 2002, Amazon launched Amazon Web Services (AWS) to provide storage and computing resources over the internet. In 2006, Amazon introduced the Elastic Compute Cloud (EC2) service, making scalable computing power available to the public.

**Computing** is the process of using computers (or machines) to process, store, and analyze data to produce useful results.

## What is Cloud Computing?
Cloud computing means using computers, storage, and services over the internet instead of buying your own hardware.

### Advantages of CLoud Computing
The following are some of the benefits of using cloud computing:

- It is easier to get backup in cloud.
- It allows us easy and quick access stored information anywhere and anytime.
- It allows us to access data via mobile.
- It reduces both hardware and Software cost, and it is easily maintainable.
- One of the biggest advantage of Cloud Computing is Database Security.
- Pay-as-you-go pricing model.

### Disadvantages
The following are some of disadvantages you will face while using cloud computing:

### Different layers of cloud computing or CLoud Service models
1. IaaS (Infrastructure as a Service)

- You get virtual servers, storage, and networks — like renting hardware from the cloud.
- Ex: AWS S3 – Cloud storage where you can store files like images, videos, backups.
- AWS EC2 – Virtual servers you can create and run like a computer in the cloud.
- Advantages: Full control over servers, Highly scalable, Pay only for what you use.
- Disadvantages: You must manage OS, updates, and security, Requires technical knowledge.

2. PaaS (Platform as a Service)
Meaning: Provides a ready-made platform to build, run, and deploy apps without managing servers. [manage only application and data]
- Examples Explained:
- AWS Elastic Beanstalk – You upload your code, and AWS handles servers, scaling, and deployment for you.
- Azure App Service – Platform to host web apps without managing infrastructure.
- AWS Lambda – Runs your code automatically without servers (serverless).
- Advantages: No server management, Faster development, Easy to deploy applications.
- Disadvantages: Less control over environment, Limited customization.

3. SaaS (Software as a Service)
- Meaning: Ready-made applications available online.
- A common example is Google Docs. You can write and share documents online without downloading any software.
- Gmail – Email service you use through the browser; no installation needed.
- Advantages: No installation needed, Accessible from anywhere, Automatic updates.
- Disadvantages: Limited control over features, Data security depends on provider.

Note: On-premise means the company keeps and manages its own servers, hardware, and software inside its own building instead of using cloud services.

#### What is Virtualization?
Virtualization is the process of creating virtual versions of physical resources like servers or storage, allowing multiple systems to run on a single physical machine using a hypervisor.

Imagine you have one powerful computer, but you want to run Windows, Linux, and another Linux server. Instead of buying 3 computers, virtualization lets you run all 3 inside one machine as separate virtual systems.

## Public vs Private Cloud?

1. Public cloud: 
- Cloud resources are owned and managed by a third-party provider and shared with multiple customers.
- Pros: Cost-effective (pay only for what you use), Highly scalable, No hardware maintenance, Easy to set up and start.
- Cons: Less control over infrastructure, Data security concerns for sensitive workloads.
- Examples: AWS, Azure, Google Cloud.

2. Private cloud:
- Cloud resources are owned and managed by a single organization and not shared with multiple customers.
- Pros: Full control over infrastructure, Data security for sensitive workloads, Customizable for specific needs.
- Cons: Higher costs (own hardware, maintenance, and support), Limited scalability, Requires dedicated resources.
- Examples: On-premise servers, dedicated data centers, VMware private cloud, OpenStack, AWS Outposts.

### Hybrid Cloud:

- Combines public and private cloud resources to create a flexible, cost-effective solution.
- Pros: Flexibility of public cloud, Security of private cloud, Cost-effective.
- Cons: Complexity of managing both types of infrastructure, Data synchronization challenges.
- Examples: AWS Outposts, Azure Stack, VMware vSphere with AWS.

### Multi-Cloud:
- Uses multiple cloud providers to meet different needs at once.
- Pros: Avoid vendor lock-in, Better flexibility, Cost-effective.
- Cons: Complexity of managing multiple providers, Data synchronization challenges.
- Examples: AWS, Azure, Google Cloud.

### Community Cloud:

- Cloud resources are owned and managed by a community of users and shared with multiple customers.
- Pros: Cost-effective, Highly scalable, No hardware maintenance, Easy to set up and start.
- Cons: Less control over infrastructure, Data security concerns for sensitive workloads. More expensive than public cloud.
- Examples: OpenStack, VMware vSphere with AWS.

## Why is public cloud more popular than private cloud?

Public cloud is more popular because it is cost-effective, highly scalable, easy to deploy, and requires no infrastructure maintenance, whereas private cloud involves high cost and management overhead.

## AWS?
Amazon Web Services (AWS) was started in 2006 to help companies avoid the high cost and effort of buying and managing their servers. Before AWS, businesses had to set up physical computers and storage to run websites or apps, which took time and money. AWS came into the market to solve this problem by offering these resources over the internet.

AWS offers over 200 services including:
- Compute → EC2[Elastic Compute Cloud] (virtual servers), Lambda (run code without server)
- Storage → S3 (object storage), EBS (disk for EC2)
- Database → RDS, DynamoDB
- Networking → VPC, Subnets, Security Groups
- IAM → Secure access control
- From startups to large enterprises like Netflix, Airbnb, and NASA, AWS is widely adopted for its flexibility, scalability, and security.

### Aws Fundamentals [Core Concepts]
1. **Regions & Availability Zones**: AWS divides its global infrastructure into regions and multiple AZs to ensure high availability and low latency.

2. **IAM (Identity and Access Management)**: Controls access by managing users, roles, and permissions securely.
3. **EC2 (Elastic Compute Cloud)**: Provides scalable virtual servers for running applications.
4. **S3 (Simple Storage Service)**: Offers secure, scalable object storage for files, backups, and static content.
5. **VPC (Virtual Private Cloud)**: Lets you create your own private network with full control over subnets, routing, and security.

### Top Reasons to Choose AWS for DevOps

- **Automation** – AWS automates deployments with tools like CodePipeline and CloudFormation.
- **Easy CI/CD** – Built-in services make CI/CD pipelines simple and fast.
- **Scalability** – Automatically scales resources based on demand.
- **Tool Integration** – Works smoothly with Jenkins, GitHub, Docker, Kubernetes, etc.
- **Strong Security** – IAM, KMS, and VPC provide secure access and protection.
- **Global Reach** – Many regions ensure low-latency deployments worldwide.
- **Cost-Effective** – Pay only for what you use, ideal for DevOps workflows.
- **Powerful Monitoring** – CloudWatch, CloudTrail, and X-Ray offer complete visibility.
- **Container Support** – ECS and EKS make container orchestration easy.
- **High Availability** – Multi-AZ architecture ensures uptime and reliability.

## Trends of people moving back to Private cloud?

“There is a growing trend called **cloud repatriation** [Repatriation is the process of moving workloads or data from public cloud back to private infrastructure or on-premise environments], where organizations are moving some workloads from public cloud back to private or on-premise environments. However, this doesn’t mean companies are abandoning cloud—instead, they are adopting hybrid cloud strategies.

This shift is mainly driven by cost optimization, data security, compliance requirements, and performance needs. Most companies now choose a hybrid model, keeping sensitive workloads in private cloud and scalable workloads in public cloud."

## AWS global Infrastructure:

**1. Regions**

- A Region is a geographical area (e.g., us-east-1, ap-south-1 (Mumbai)).
- Each Region is independent and isolated from others.
- You choose a Region based on: [vvp]
    - Reduced Latency [Latency is the time taken for data to travel from source to destination.]
    - Compliance/data residency -> [data never leaves a region without permission]
    - Cost
    - Service availability
- 👉 Example: India users often choose ap-south-1 (Mumbai).

**2. Availability Zones (AZs)**

- Each Region has 2 or more Availability Zones.
- An AZ = one or more physically separate data centers.
- Connected via high-speed, low-latency private fiber.
- Designed for fault tolerance.
- Separate from each other, so that theyre isolated from disasters.
- It composed of one or more discrete data centers with redundant power, networking, and connectivity, and are used to deploy infrastructure. [defination]
- 👉 Best practice: Deploy apps across multiple AZs.

**3. Data Centers**

- Physical buildings with servers, storage, and networking.
AWS does not disclose exact locations for security reasons.
Built with:
- Redundant power
- Cooling
- Networking
- Physical security
