## AWS Security using Security Groups and NACL

### 🔹 1. What is a Security Group?

A **Security Group** acts as a virtual firewall at the **instance level**.

It controls inbound and outbound traffic to AWS resources like EC2.

#### Key Points:
- Operates at **instance level**
- **Stateful**
  - If inbound traffic is allowed, response is automatically allowed
- Supports **only Allow rules**
- Evaluates **all rules together**

#### Example:
If port 22 (SSH) is allowed from your IP, you can connect to the instance, and the return traffic is automatically allowed.

### 🔹 2. What is NACL (Network ACL)?

A **Network ACL (NACL)[Network Access Control List]** acts as a firewall at the **subnet level**.

It controls traffic entering and leaving a subnet.

#### Key Points:
- Operates at **subnet level**
- **Stateless**
  - Must explicitly allow both inbound and outbound traffic
- Supports **Allow and Deny rules**
- Rules are processed in **numerical order (lowest first)**

#### Example:
If inbound port 80 is allowed, outbound ephemeral ports (1024–65535) must also be allowed for proper communication.

### 🔥 3. Key Differences

| Feature            | Security Group            | NACL                          |
|-------------------|--------------------------|-------------------------------|
| Level             | Instance level           | Subnet level                  |
| State             | Stateful                 | Stateless                     |
| Rules             | Allow only               | Allow & Deny                 |
| Rule Evaluation   | All rules evaluated      | Processed in order           |
| Return Traffic    | Automatically allowed    | Must be explicitly allowed   |

### 🧠 4. How They Work Together

- **Security Group** → Instance-level protection  
- **NACL** → Subnet-level protection  

> Both must allow traffic for it to pass.

---

### 🎯 5. Real-Time Scenario

Suppose you have a web server in a public subnet:

#### NACL:
- Allow inbound HTTP (port 80) from anywhere
- Allow outbound ephemeral ports (1024–65535)

#### Security Group:
- Allow inbound HTTP (port 80) from `0.0.0.0/0`

If either blocks traffic, the request will fail.

### ⚡ 6. When to Use What?

#### Use Security Groups:
- Instance-level control
- Daily usage
- Most common scenarios

#### Use NACL:
- Additional security layer
- Blocking specific IPs (Deny rules)

### 🚀 7. One-Line Summary

> Security Groups are stateful instance-level firewalls, whereas NACLs are stateless subnet-level firewalls with both allow and deny rules.

### 💡 Pro Tip

Security Groups are sufficient for most use cases.  
NACLs are mainly used for coarse-grained control or explicit deny scenarios.

### Story of port 25
Port 25 is used for SMTP (Simple Mail Transfer Protocol) which is used to send and receive emails. We can't use this port for security reasons.

Port 25 is used for SMTP, but it is restricted in modern environments due to spam and security risks. Instead, secure ports like 587 or 465 are used with authentication and encryption.
