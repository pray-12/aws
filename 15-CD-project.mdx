**AWS CodeDeploy** is an AWS service used to automatically deploy your application code to servers or services.


AWS CodeDeploy is a deployment service that automates application deployments to EC2, ECS, and Lambda with support for in-place and blue-green deployment strategies

## How CodeDeploy works
1. Code is stored in a repository (like GitHub).
2. Code is deployed to a server or service.

### Create a EC2 instance
1. Go to the AWS Management Console.
2. Navigate to the EC2 service.
3. Click on "Instances".
4. Click on "Launch Instance".
5. Do the configuration and make sure in network settings vpc is selected as "project" and subnet is selected as "public-subnet". enable public ip address.
6. Add a tag with key as "Name" and value as "MyEC2Instance".
7. Create instance.

### Logged in to the EC2 instance
1. ssh -i ~/Downloads/my-key-pair.pem ubuntu@ec2-instance-public-ip
2. sudo apt update
3. Run commands from here: https://docs.aws.amazon.com/codedeploy/latest/userguide/codedeploy-agent-operations-install-ubuntu.html to install the CodeDeploy agent. Why we are doing this? Because CodeDeploy agent is responsible for deploying the application to the EC2 instance.

### IAM Role for CodeDeploy
1. Go to the IAM service.
2. Click on "Roles".
3. Click on "Create role".
4. Select the service as "CodeDeploy".
5. Click on "Create role".
6. Add a tag with key as "Name" and value as "CodeDeployRole" &"EC2FullAccess".
7. Create role.
8. Now attach the role to the EC2 instance.
    - Action > Security > Modify IAM role>update IAM role
9. Restart the agent by running the command: sudo service codedeploy-agent restart

### Create a CodeDeploy application
1. Go to the CodeDeploy service.
2. Click on "Applications".
3. Click on "Create application".
4. Click on "Create deployment group".
    -  Give the group name as "MyDeploymentGroup".
    -  Service role as "CodeDeployRole".
    -  Environment configuration: Amazon EC2 instances.
       - Key : Name , Value : MyEC2Instance [1 unique match instance required]
    - Load balancer configuration: None

5. Create a appspec.yml file that contains the configuration for the application. It is a YAML file that contains the configuration for the application. It is used to define the application and the deployment group. [root repository is the repository that contains the code that you want to deploy.]

6. Go to application>deployment groups>MyDeploymentGroup>Deployment configuration>Create a new deployment.
7.  Connect to the GitHub repository and add the commit id of the code that you want to deploy.

```yaml
version: 0.0
os: linux

hooks:
  ApplicationStop:
    - location: scripts/stop_container.sh
      timeout: 300
      runas: root
  AfterInstall:
    - location: scripts/start_container.sh
      timeout: 300
      runas: root
```

scripts/stop_container.sh and scripts/start_container.sh are the scripts that you want to run before and after the deployment.
```bash
#!/bin/bash
set -e # This is used to exit the script if any command fails.
echo "Stopping the container..."

container_id=`docker ps | awk -F " " '{print $1}'
docker rm -f $container_id
echo "Container stopped and removed..."
```

```bash
#!/bin/bash
set -e # This is used to exit the script if any command fails.

# Pull the Docker image from Docker Hub
echo "Pulling the Docker image from Docker Hub..."

docker pull <image-name>
docker run -d -p 5000:5000 <image-name> # This is used to run the Docker image as a container on port 5000.

# Run the Docker image as a container
echo "Running the Docker image as a container..."
```
